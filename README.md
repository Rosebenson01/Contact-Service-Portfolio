Module 8
1) Ensuring that my code, program, or software is functional and secure I do a combination of thorough testing, careful coding practices, and reviewing to make sure I am inputting the intended code. I make sure to write and run tests, to verify that each part of the program works as intended. Peer code reviews help catch errors, logic issues, or security vulnerabilities that I might miss. I also follow secure coding practices, like avoiding hardcoded sensitive data and using encryption when needed. Keeping dependencies updated and monitoring for security vulnerabilities ensures that the software remains secure over time.

2) Interpreting user needs and incorporating them into a program starts with analyzing requirements. I communicate with users to understand their goals, challenges, and expectations for the software. From these conversations, I identify patterns, prioritize essential features, and document use cases that shows how users will interact with the program. I also incorporate feedback throughout development, using prototypes to confirm that the software meets real-world needs rather than assumptions. This iterative approach ensures that the program remains aligned with user expectations and provides meaningful value.

3) I start by defining the purpose, scope, and main features of the program, then outline a high-level architecture showing how components interact. Choosing appropriate programming languages, frameworks, and tools helps ensure the software can meet its goals effectively. I focus on modular design, breaking the program into manageable components, and consider how data flows through the system to optimize performance and usability.
